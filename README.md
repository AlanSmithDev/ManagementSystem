# yudao-cloud 芋道管理系统

企业级微服务架构系统，Spring Cloud Alibaba + Vue3 + UniApp

## 项目详细介绍

### 1.1 项目定位

**yudao-cloud（芋道）** 是一套面向 **学习与实战** 的企业级微服务架构系统，采用 Spring Cloud Alibaba 2021 + Vue3 + UniApp 技术栈。

本项目旨在为 Java 开发者提供一套 **开箱即用的企业级解决方案**，代码完全开源，个人与企业可 **100% 免费使用**。

| 定位人群 | 学习内容 | 收获 |
|:---|:---|:---|
| **Java 初学者** | Spring Boot 基础、Web 开发、数据库操作 | 掌握企业级项目开发流程 |
| **中级开发者** | Spring Cloud 微服务、分布式、缓存、消息队列 | 成为微服务开发工程师 |
| **高级开发者** | 架构设计、高并发、性能优化、微服务治理 | 向架构师方向发展 |
| **架构师** | 参考设计方案、快速搭建系统 | 提升开发效率 80%+ |

## 技术栈详解

### 后端技术栈（25+核心技术）

本项目采用 Spring Cloud Alibaba 微服务技术栈，覆盖企业级开发的方方面面。

#### 核心框架层

| 技术 | 版本 | 说明 | 地位 |
|:---|:---:|:---|:---|
| **Spring Boot** | 2.7.18 | 基础框架，约定优于配置 | ⭐ 必备基础 |
| **Spring Cloud Alibaba** | 2021.0.4.0 | 阿里巴巴微服务生态圈 | ⭐ 核心架构 |
| **Spring Cloud** | 2022.0.0 | 微服务标准规范 | ⭐ 核心架构 |

#### 服务治理层

| 技术 | 版本 | 说明 | 核心能力 |
|:---|:---:|:---|:---|
| **Nacos** | 2.3.2 | 服务注册中心 + 配置中心 | 服务发现、配置管理、多环境隔离 |
| **Spring Cloud Gateway** | 3.4.1 | API 网关 | 路由转发、限流熔断、统一认证 |
| **Sentinel** | 1.8.6 | 流量控制组件 | 流量限制、熔断降级、系统自适应保护 |
| **LoadBalancer** | 3.4.1 | 客户端负载均衡 | 轮询、随机、加权策略 |

## 实现细节

### 模块划分原则

本项目采用 **领域驱动设计（DDD）** 思想，按照业务领域划分模块，每个模块高内聚、低耦合，可独立开发、测试、部署。

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              模块划分原则详解                                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  【原则一：高内聚低耦合】                                                          │
│  ┌───────────────────────────────────────────────────────────────────────────┐  │
│  │  • 每个模块职责单一，只负责一个业务领域                                        │  │
│  │  • 模块间通过 API 接口通信，不直接依赖对方实现类                               │  │
│  │  • 模块内聚性高，相关逻辑集中在一起                                            │  │
│  │  • 模块间耦合度低，接口稳定，实现可替换                                        │  │
│  └───────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │
│  【原则二：前后端分离】                                                            │
│  ┌───────────────────────────────────────────────────────────────────────────┐  │
│  │  • 后端提供 RESTful API，通过 JSON 交互                                      │  │
│  │  • 前端完全独立，可单独开发、测试、部署                                        │  │
│  │  • 前后端通过接口约定通信，前后端可并行开发                                    │  │
│  │  • 支持多端接入：Web、iOS、Android、微信小程序                                │  │
│  └───────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │
│  【原则三：领域驱动设计】                                                          │
│  ┌───────────────────────────────────────────────────────────────────────────┐  │
│  │  • 按照业务领域划分模块（System/BPM/Mall/CRM/ERP/IoT）                      │  │
│  │  • 每个模块包含完整的业务功能（Controller/Service/Repository）                  │  │
│  │  • 模块间通过 API 层暴露服务，隐藏内部实现                                     │  │
│  │  • 统一使用 Maven/Gradle 管理模块依赖                                        │  │
│  └───────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │
│  【原则四：技术复用与分层】                                                         │
│  ┌───────────────────────────────────────────────────────────────────────────┐  │
│  │  • 通用能力下沉到 framework 层，供所有模块复用                                │  │
│  │  • 框架层包含：安全、缓存、消息、任务调度、监控等                              │  │
│  │  • 业务代码只关注业务逻辑，基础设施代码统一抽象                                │  │
│  └───────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

**配置示例：**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfiguration {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            // 禁用 CSRF
            .csrf(csrf -> csrf.disable())
            // 登录入口
            .authorizeHttpRequests(c -> c
                // 匿名接口：登录、验证码、文档等
                .requestMatchers(
                    "/auth/login",
                    "/auth/captcha",
                    "/doc.html",
                    "/swagger-ui/**",
                    "/v3/api-docs/**",
                    "/favicon.ico"
                ).permitAll()
                // 公共路径
                .requestMatchers("/webjars/**").permitAll()
                // 其他请求需要认证
                .anyRequest().authenticated()
            )
            // Session 管理
            .sessionManagement(c -> c
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            )
            // 禁用表单登录
            .formLogin(form -> form.disable())
            // 禁用 HTTP Basic
            .httpBasic(basic -> basic.disable());
        
        return http.build();
    }
}
```

```java
@Service
public class UserCacheServiceImpl implements UserCacheService {

    private static final String USER_KEY = "user:{}";
    private static final long CACHE_EXPIRE_TIME = 30 * 60; // 30分钟

    @Autowired
    private StringRedisTemplate redisTemplate;

    @Override
    public AdminUserDO getUser(Long userId) {
        String key = MessageFormat.format(USER_KEY, userId);
        
        // 1. 先查缓存
        String cached = redisTemplate.opsForValue().get(key);
        if (StringUtils.hasText(cached)) {
            return JSON.parseObject(cached, AdminUserDO.class);
        }
        
        // 2. 缓存未命中，查数据库
        AdminUserDO user = adminUserMapper.selectById(userId);
        if (user != null) {
            // 3. 写入缓存
            redisTemplate.opsForValue().set(key, JSON.toJSONString(user), 
                CACHE_EXPIRE_TIME, TimeUnit.SECONDS);
        }
        
        return user;
    }

    @Override
    public void evictUser(Long userId) {
        String key = MessageFormat.format(USER_KEY, userId);
        redisTemplate.delete(key);
    }
}
```

```java
@Service
public class OrderMessageProducer {

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    public void sendOrderPaidMessage(Long orderId, BigDecimal amount) {
        OrderPaidMsg msg = new OrderPaidMsg();
        msg.setOrderId(orderId);
        msg.setAmount(amount);
        msg.setPaidTime(new Date());
        
        // 发送事务消息
        rocketMQTemplate.sendMessageInTransaction(
            "order-topic:paid",
            MessageBuilder.withPayload(msg)
                .setHeader("orderId", orderId)
                .build(),
            orderId  // 事务上下文参数
        );
    }
}

@RocketMQMessageListener(
    topic = "order-topic",
    consumerGroup = "order-consumer-group",
    selectorExpression = "paid"
)
public class OrderPaidConsumer implements RocketMQListener<OrderPaidMsg> {

    @Override
    public void onMessage(OrderPaidMsg msg) {
        log.info("收到订单支付消息: orderId={}", msg.getOrderId());
        
        try {
            // 1. 扣除库存
            stockService.deductStock(msg.getOrderId());
            
            // 2. 发送发货通知
            deliveryService.sendDeliveryNotice(msg.getOrderId());
            
            // 3. 更新订单状态
            orderService.updateOrderStatus(msg.getOrderId(), OrderStatus.DELIVERED);
            
            log.info("订单支付消息处理成功: orderId={}", msg.getOrderId());
        } catch (Exception e) {
            log.error("订单支付消息处理失败: orderId={}", msg.getOrderId(), e);
            throw new RuntimeException("消息处理失败，需要重试");
        }
    }
}
```

**MyBatis Plus 乐观锁实现：**

```java
// 实体类添加 @Version 注解
@TableName("product")
public class ProductDO extends TenantBaseDO {
    @TableId
    private Long id;
    
    private String name;
    private Integer stock;
    
    @Version  // 乐观锁版本字段
    private Integer version;
}

// Service 层使用
public void deductStock(Long productId, Integer count) {
    // MyBatis Plus 会自动处理版本号
    ProductDO product = productMapper.selectById(productId);
    
    if (product.getStock() < count) {
        throw new BizException("库存不足");
    }
    
    // UPDATE product SET stock = stock - #{count}, version = version + 1
    // WHERE id = #{id} AND version = #{version}
    LambdaQueryWrapper<ProductDO> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(ProductDO::getId, productId)
           .eq(ProductDO::getVersion, product.getVersion());
    
    ProductDO update = new ProductDO();
    update.setStock(product.getStock() - count);
    
    int rows = productMapper.update(update, wrapper);
    if (rows == 0) {
        throw new BizException("并发更新失败，请重试");
    }
}
```

### 用户商城小程序（yudao-mall-uniapp）

面向 C 端用户提供商城购物流程，支持 **微信小程序/支付宝小程序/H5** 多端运行。
